2024 SuiOverflow Hackathon 參賽心得
Jun 7, 2024

我的背景以及為何我會參加 Hackathon?

其實早在大約 2019 年時我從海外離職回到台灣時，就有過想要參加黑客松的想法了。
但那個時候因為剛回台灣不熟悉就業環境，我給自己的目標其實很簡單：

  1. 在台灣找到一個技術端的工作。
  2. 多碰一些跟開發相關的事情。

上面這兩個目標在 2019 年很快的都被實現了，回到台灣我繼續做測試工程師、也在原本熟悉的測試領域上引入了一些自動化測試的框架，觸碰了一些代碼、做著做著又過了一個 5 年，中間遇到疫情等等重大事件，參加黑客松這個想法也不了了之。

曾經我一直以為我會繼續做測試下去直到老死。

雖然潛意識中是心有不甘的，因為一直以來我總有要 Build 屬於自己的專案的念頭。直到去年底離職後，才開始有時間和空間直面自己心裡所想做的事情。後來因緣際會下在 Sui 所舉辦的開發者活動中認識了現在的隊友，於是開始了本次的 Hackathon 之旅。

參加 Hackathon 最大的收獲

收獲最多莫過於心理上的成就感，畢竟終於實現了我想要「做一個自己的產品」的想法。

中間的過程其實滿辛苦的，我們這一組雖說有 5 個人，但有寫程式經驗的只有我和另外一位工程師。我過去是測試的角色，產品、PM、開發的角色都略懂但是並不精通。

這次黑客松團隊中，我主要的角色是編寫合約並串接到前端 dApp，在不熟悉智能合約語言的情況下從 5/10 左右才開始認真學習 Move，並在 5/31 號將完成的出版交付出去，學習曲線其實是非常陡峭的，大概在交件截止前的一週，連續幾天都是寫到天亮才休息。

雖然非常辛苦，但在中間學習到的東西也不少：

  1. 用 Move 語言編寫一個任務版的 CRUD 功能。
  2. 將鏈上的合約函數串到 next.js 框架做的 dApp 前端，完成業務邏輯的閉環。
  3. 自己設計產品的使用流程、世界觀、特色功能、前端介面的美術風格。
  4. 做簡報、寫 README 來表達專案的特色和理念。

雖然最終的產品功能比較陽春、很多預想要有的功能也沒有實現；但對我來說這就是好的開始了。
對於這個專案我還有很多其他的想法想要實現，不管這次有沒有獲得獎金、名次；我都會繼續發展下去，直到我發現更好玩的 IDEA 並決定投入為止。

Project Info

目前在 SuiDevnet 上有部署合約、尚未更新 Testnet，Devnet 每週會清理數據，因此可能會無法使用。專案更新至 Testnet 時會更新說明。

我們的公開任務模型設計具有以下特點：

  1. 追求透明和開放：任何人都可以提交他們的工作，無需獲得許可。認證通過的作品將會使用 Sui 的 Freeze Object 功能在鏈上凍結和保存。
  2. 鏈上合約與直接獎金分配：參與者可以在開始任務前確認獎金的存在並通過智能合約的自動獎勵分配機制確保分配金額的準確性。
